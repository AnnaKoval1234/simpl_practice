1) Почему побитовые операции в примерах ниже не меняют число? Что они делают внутри?

alert( 123 ^ 0 ); // 123
alert( 0 ^ 123 ); // 123
alert( ~~123 ); // 123

Ответ: в случае с исключающим ИЛИ с 0 результатом будет исходное число. 
В случае с побитовым НЕ число сначала инвертируется, затем инвертируется обратно.

3) Верно ли, что для любых a и b выполняются равенства ниже?

(a ^ b) == (b ^ a)
(a & b) == (b & a)
(a | b) == (b | a)

Иными словами, при перемене мест – всегда ли результат останется тем же?

Ответ: да, результат остается тем же.

4) Почему результат второго alert такой странный?

alert( 123456789 ^ 0 ); // 123456789
alert( 12345678912345 ^ 0 ); // 1942903641

Ответ: вероятно, произошло переполнение, т.к. второе число занимает больше 32 бит.
